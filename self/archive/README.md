# self/archive/

> **목적**: 자기 정의(self-definition.md)에서 격리된 항목의 영구 보관소
> **정책**: never_delete

---

## never_delete의 의미

자기 정의에서 어떤 항목이 더 이상 유효하지 않게 되어도, 하네스는 그것을 **삭제하지 않습니다.** 대신 여기로 **이동(Tier 격리)** 합니다. 이는 W.A.Y? 하네스의 핵심 데이터 라이프사이클 원칙입니다.

- **삭제 ≠ 폐기**: 폐기 결정도 "Tier 이동"으로 이행됩니다. 흔적 자체는 영구히 남습니다.
- **이유**: (1) 과거 선언·관찰의 변천을 추적할 수 있어야 자기 정의의 진화 경로가 보존됩니다. (2) 약화·폐기된 항목이 나중에 다시 도전받을 때 비교 기준이 됩니다.
- **이동 트리거**: 새 데이터가 기존 항목에 도전 + Critical 결재 통과 시에만 여기로 이동합니다. 자동 이동은 없습니다 (Push 기반 신선도, 능동적 stale 검출 없음).

---

## 구조

이동된 항목은 `self/changelog.md`에 "이동(Archive)" 유형으로 기록된 뒤, 이 폴더에 보관됩니다.

`.gitkeep`은 빈 폴더 추적을 위해 유지합니다 (삭제 금지).
