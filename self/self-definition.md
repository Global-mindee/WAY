## 이 파일이 무엇인가

이 파일은 W.A.Y? 하네스가 운영자(사용자)를 이해하기 위해 보관하는 **자기 정의(Self-Definition)** 스키마입니다. 5개 영역(A 정체성 / B 지식 / C 사고방식 / D 작업패턴 / E 선호)에 걸쳐 운영자가 누구이고, 무엇을 알고, 어떻게 생각하고 일하며, 무엇을 선호하는지를 표로 누적합니다.

빈 상태로 배포되며, **온보딩(sde-extractor)이 당신의 메모리·git 히스토리·설정 파일을 분석해 자동으로 채웁니다.** 아래 표는 골격과 placeholder 행 1개씩만 담고 있습니다.

## 0-1. Declared vs Observed vs Gap

각 영역은 세 갈래로 데이터를 구분합니다. 이 구분은 하네스가 "선언만 있고 실제는 다른" 상태를 충돌(conflict)로 표면화하는 근거가 됩니다.

- **Declared(선언)**: 사용자가 직접 명시한 내용.
- **Observed(관찰)**: 환경 데이터(git·로그·설정·메모리)에서 추출된 내용.
- **Gap(차이)**: 선언과 관찰의 불일치. Gap이 누적되면 `self/conflicts.md`에 결재 대기 항목으로 등록됩니다.

> 각 영역 표 아래에 "Declared / Observed / Gap" 3구분 안내를 1줄씩 남깁니다.

## 온보딩 안내

이 파일은 온보딩(`sde-extractor`)이 당신의 메모리·git·설정을 분석해 자동으로 채웁니다. 채워진 뒤에는 다음 메커니즘이 작동합니다.

- **변화 추적**: 자기 정의의 Critical·Notable 변경은 `self/changelog.md`에 누적됩니다.
- **충돌 표면화**: Declared vs Observed의 Gap은 `self/conflicts.md`에 결재 대기 항목으로 등록됩니다.
- **소규모 변경**: Background 등급 잡음 변경은 `self/background-log.md`에 1줄씩 누적됩니다(never_delete).
- **정본 관계**: MEMORY 출처의 정본은 Claude Code 내장 auto-memory이며, 본 자기 정의는 그와 별도로 운영자 모델을 보관합니다.
