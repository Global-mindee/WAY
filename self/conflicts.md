# self/conflicts.md

> **목적**: 자기 정의(self-definition.md) 내 미해결 충돌 항목 (사용자 결재 대기 큐)
> **분류 3종**: 활성 전환 (Active Transition) / 정체 (Stagnation) / 모순 (Contradiction)
> **자동 분류 정책**: 모순(Contradiction)만 시스템 자동 분류 + Critical 결재 — 활성 전환·정체는 사용자가 직접 분류
> **출처**: `harness-blueprint.md` (2) Item 3 + `CLAUDE.md` 메모리 라이프사이클 절

---

## 이 파일이 무엇인가

자기 정의의 **선언(Declared) vs 관찰(Observed) Gap**이 누적되면 여기에 충돌 항목으로 등록됩니다. 이 큐는 **무기한 대기**하며 자동 만료·자동 해소되지 않습니다. 사용자가 체크박스로 직접 결재해야 항목이 닫힙니다.

---

## 분류 3종 설명

| 분류 | 의미 | 자동/수동 |
|------|------|----------|
| 활성 전환 (Active Transition) | 의도된 변화가 진행 중 — 선언이 앞서고 환경이 따라오는 중 | 사용자 분류 |
| 정체 (Stagnation) | 선언만 있고 오래 진척 없음 — 재검토 필요 | 사용자 분류 |
| 모순 (Contradiction) | 선언과 관찰이 양립 불가 | 시스템 자동 분류 + Critical 결재 |

---

## 형식

```
## ☐ 충돌 N: 제목
- **분류**: 활성 전환 / 정체 / 모순
- **영역**: A·B·C·D·E
- **선언적 상태 (Declared)**: 사용자 명시한 내용
- **관찰적 상태 (Observed)**: 환경에서 추출된 내용
- **Gap**: 양자 간 차이 요약
- **결재 옵션**:
  - [ ] 선언 채택 (관찰 갱신)
  - [ ] 관찰 채택 (선언 갱신)
  - [ ] Active Transition으로 분류 (진행 중)
  - [ ] 정체로 분류 (재검토 필요)
```

### ☐ 충돌 (예시): 활동 우선순위 (선언 vs 관찰)

- **분류**: 활성 전환 (Active Transition)
- **영역**: D 작업 패턴
- **선언적 상태**: 사용자가 새 프로젝트를 메인 활동으로 선언
- **관찰적 상태**: 환경 데이터(git·로그)는 아직 기존 활동 중심으로 기록
- **Gap**: 선언만 있고 환경 데이터 미반영
- **결재 옵션**:
  - [ ] Active Transition 유지 (다음 SDE 트리거 후 재평가)
  - [ ] 환경 데이터 강제 갱신 (수동)
  - [ ] 선언 취소 (기존 우선순위 유지)
