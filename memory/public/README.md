# memory/public/

> **Tier**: Public (3-Tier 메모리 중 1단계)
> **접근**: 명시 호출 (결재 통과 지식의 승격 보관소)

---

## 3-Tier 메모리 구조

W.A.Y? 하네스의 `memory/`는 **결재 통과 지식의 승격 보관소**입니다. 전부 명시 호출만으로 접근하며, 자동 인젝션은 없습니다.

| Tier | 위치 | 접근 |
|------|------|------|
| **Public** | `memory/public/` (여기) | 명시 호출 (결재 통과 지식 승격분) |
| Quarantine | `memory/quarantine/` | 명시 호출만 |
| Archive | `memory/archive/` | 명시 호출만, never_delete |

---

## 4타입 분류

메모리 항목은 4타입으로 분류합니다.

| 타입 | 의미 |
|------|------|
| User | 사용자 자신에 관한 지식 (선호·정체성 보강 등) |
| Project | 특정 프로젝트의 결정·맥락 (메타 정보만, 운영 데이터 격리) |
| Feedback | 작동 관찰·교훈에서 승격된 지식 |
| Reference | 외부 출처·자료 인용 보관 |

---

## auto-memory 정본 관계

**MEMORY 출처의 정본은 Claude Code 내장 auto-memory**입니다(자동 작동). 하네스 `memory/` 3-Tier는 그와 별개로, **결재를 통과한 지식만 승격해 보관하는 곳**입니다.

- `[M:]` 마커는 스토어를 구분해 표기합니다: `[M: auto/<이름>]` (내장) / `[M: harness/<tier>/<이름>]` (여기).
- 자동 인젝션 약속은 없습니다 — 전부 명시 호출.

## Push 기반 신선도

- 능동적 stale 검출 없음.
- 새 데이터가 기존 데이터에 도전할 때만 Tier 이동.
- 데이터 부재는 stale 신호가 아닙니다.

`.gitkeep`은 빈 폴더 추적을 위해 유지합니다 (삭제 금지).
