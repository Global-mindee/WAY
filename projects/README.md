# projects/

> **목적**: 프로젝트별 메타 정보 보관 (운영 데이터 격리)
> **출처**: `CLAUDE.md` 운영 데이터 격리 절 (P6 정신)

---

## 단방향 격리 원칙

> 데이터는 새지 않되, 교훈은 수집된다.

이 폴더에는 **프로젝트별 메타 정보만** 둡니다. 실제 운영 데이터(매출·결재 내용·API 응답·고객 정보 등)는 절대 여기에 두지 않으며, 각 프로젝트의 **외부 레포에 격리**합니다.

| 둘 수 있음 (메타) | 둘 수 없음 (운영 데이터) |
|------------------|------------------------|
| 이벤트·패턴·정의·통계 | 실제 매출·금액 |
| 프로젝트 상태·단계 메모 | 고객 정보·개인정보 |
| 결정의 근거·요약 | API 원본 응답·결재 본문 |

---

## 가상 예시

```
projects/
  project-alpha/
    README.md      ← 프로젝트 개요·단계 메타만 (운영 수치 없음)
    decisions.md   ← "어떤 결정을 왜 내렸나"의 메타 요약 (금액·실명 없음)
```

> 예: "project-alpha — Phase 1 진입, 외부 레포는 별도. 본 폴더엔 진척 단계 메타만 보관" 처럼, 실제 데이터는 외부 레포를 가리키고 여기엔 포인터·메타만 남깁니다.

`.gitkeep`은 빈 폴더 추적을 위해 유지합니다 (삭제 금지).
