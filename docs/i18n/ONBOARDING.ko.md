# W.A.Y? 시작하기

> W.A.Y?를 "막 클론한 상태"에서 "당신의 개인 하네스로 가동"까지 데려가는 단계별
> 체크리스트입니다. 이것을 끝내는 데 개발자일 필요는 없습니다. 어려운 부분 — 당신의
> 맥락을 읽고 당신의 모델을 초안 작성하는 일 — 은 대신 처리됩니다. 당신의 일은 검토하고
> 승인하는 것입니다.

각 단계는 **자신이 건드리는 파일**을 명시하므로, 당신이 볼 수 없는 일이 당신의 환경에서
일어나는 일은 없습니다.

---

## 시작하기 전에

- 로컬 CLI 에이전트(Claude Code, 또는 당신의 환경을 읽을 수 있는 다른 CLI).
- 검토 단계에 잠깐의 주의 — 그곳이 **당신**을 필요로 하는 유일한 지점입니다.

하네스는 코어만으로도 작동합니다. 아래의 선택적 플러그인은 그 도달 범위를 넓힐 뿐이니,
최소 구성을 원하면 건너뛰어도 됩니다.

---

## 1단계 — 클론

당신의 CLI 에이전트가 도는 머신에 저장소를 가져옵니다.

```bash
git clone <당신의-포크-또는-이-레포-url> WAY
cd WAY
```

- **건드림**: 새 `WAY/` 작업 디렉터리. 당신의 홈이나 다른 저장소는 건드리지 않습니다.
- 저장소 루트의 `CLAUDE.md`는 모든 에이전트가 여기서 작업 시작 시 자동 로드하는 베이스
  지침입니다. 온보딩하기 위해 그것을 편집하지 않습니다.

---

## 2단계 — 플러그인 설치 (선택)

W.A.Y?는 어떤 플러그인 없이도 작동합니다. 더 강한 웹 리서치를 원한다면 두 가지를
권장합니다:

- **insane-search** — 차단/봇 보호된 출처에 대한 적응형 접근으로, 시장·웹 리서치가 조용히
  데이터를 잃지 않게 합니다. (fivetaku, MIT — `CREDITS.md` 참조.)
- **deep-research** — 다중 출처·사실 검증 리서치와 인용 종합으로, full-loop 리서치
  단계와 짝을 이룹니다. (fivetaku — `CREDITS.md` 참조.)

당신의 CLI 플러그인 명령으로 설치합니다(Claude Code의 경우 `gptaku-plugins` 마켓플레이스에서
`/plugin` 흐름). 플러그인은 보통 세션 재시작 후 활성화됩니다.

- **건드림**: 당신의 CLI 플러그인 설정(user 스코프). 하네스 파일은 건드리지 않습니다.
- **건너뛰기 친화적**: 이를 건너뛰면 하네스는 일반 웹 조회와 정직 모드로 폴백합니다.
  봇 보호된 출처의 커버리지만 잃습니다.

---

## 3단계 — 자기 정의 추출기 실행

이것이 온보딩의 핵심입니다. `sde-extractor` 스킬을 실행하세요(그 프롬프트는
`skills/05_knowledge-and-memory/sde-extractor/SKILL.md`에 있습니다). 그 프롬프트를 CLI
에이전트의 첫 메시지로 붙여 넣거나, 스킬을 직접 호출할 수 있습니다.

추출기는 접근 가능한 맥락을 순서대로 읽습니다: 이전 대화, 홈 디렉터리 설정, 하네스 파일,
활성 프로젝트 저장소, git 활동, 셸 설정, 최근 작업 디렉터리. 각 출처에서 무엇을
발견했는지(그리고 무엇에 접근할 수 없었는지) 보고합니다.

- **건드림(읽기만)**: 당신의 CLI 기억, `~/` 설정 파일, 이 하네스, 당신의 프로젝트 저장소,
  `git log`, 셸 설정. 이것들 중 무엇도 **수정하지 않습니다**.
- **씀(초안)**: `self/self-definition.md`의 초안.

---

## 4단계 — 자기 정의가 자동으로 초안 작성됨

읽은 것으로부터, 추출기는 다섯 영역 — 정체성, 지식, 사고 스타일, 작업 패턴, 선호 — 을
채우고, 모든 항목에 신뢰도 마커를 붙입니다:

- 데이터에서 직접 확인됨,
- 패턴에서 강하게 추론됨(그 패턴을 명시),
- 미상 — 당신에게 물어야 함.

그것은 미상을 추측하지 **않습니다**. 그것들을 모아, **한 번**, 묶음 질문 하나로 당신에게
묻습니다(많으면 가장 중요한 다섯 개부터). 그것들에 답하면, 당신의 답을 다시 접어 넣습니다.

- **씀**: `self/self-definition.md` (구조화된 모델).
- 모델이 시간에 따라 어떻게 바뀌는지 추적하는 동반 파일이 그 곁에 함께 있습니다:
  `self/changelog.md`, `self/conflicts.md`, `self/background-log.md`.

---

## 5단계 — 검토하고 승인

`self/self-definition.md`를 열고 읽으세요. 이것이 당신의 판단을 요구하는 유일한 단계입니다
— 하네스가 이 모델을 근거로 행동할 것이므로, 그것은 진짜 당신을 반영해야 합니다.

- 잘못 추론된 것을 바로잡으세요. 추출기가 미상으로 표시한 항목을 채우거나 고치세요.
- 맞아 보이면 승인하세요.

자기 정의에 대한 중요한 변경은, 조용한 자기 재작성이 아니라 이 승인을 거칩니다 — 지금도,
나중에 모델이 진화할 때도.

- **건드림**: `self/self-definition.md`(당신의 편집), 그리고 사인오프가 필요한 모든 것에
  대한 `decisions/pending.md`의 결재/결정 흔적.

---

## 6단계 — 하네스가 가동됨

여기서부터, 당신의 에이전트가 이 저장소에서 작업을 시작할 때마다 다음을 순서대로
자동 로드합니다:

1. `self/self-definition.md` — 당신의 현재 정체성과 작업 패턴.
2. `decisions/pending.md` — 당신을 기다리는 모든 결재(첫 응답에 표면화됨).
3. `rules/` — 필요에 따라 호출되는 운영 규칙.
4. `skills/USAGE-GUIDE.md` + `skills/INDEX.md` — 당신의 요청에 맞는 스킬을 고르기 위해.
5. `agents/USAGE-GUIDE.md` + `agents/INDEX.md` — 맞는 서브 에이전트를 고르기 위해.

이제 신뢰 기계장치가 자동으로 돌아갑니다: 출처 검증 출력(SVOP), Mode Toggle, 다섯 가지
정직 모드, Best-of-N 검증을 갖춘 Critical Path, 그리고 출처 신뢰도 등급화.

---

## 7단계 — full-loop 써 보기 (다단계 작업이 생겼을 때)

리서치 + 계획 + 빌드 + 검증이 필요한 무엇이든, **full-loop**
(`skills/07_orchestration/full-loop/SKILL.md`)에 맡기세요. "이걸 끝까지 실행해" 또는
"이거 full-loop 해"처럼 말하면, 그것이 8단계 파이프라인을 몰아갑니다 — 인간 게이트(계획
승인, 외부 영향, 재시도 소진)에서 멈추고, 다른 어디에서도 멈추지 않습니다.

- **건드림**: 루프가 중단을 견디도록 하는 로컬 루프 상태 파일
  (`.claude/full-loop-state.local.json`), 그리고 작업 자체가 겨냥하는 무엇이든.
- 한 줄 수정이나 빠른 질문에는 full-loop를 **쓰지 마세요** — 그것은 그렇다고 말해 주고
  그냥 직접 답할 것입니다.

---

## 좋은 상태란

- `self/self-definition.md`가 존재하고, 당신이 검토했으며, 미상이 추측되지 않고 해소되어
  있습니다.
- 저장소에서 당신 에이전트의 첫 응답이 대기 중인 결재를 표면화합니다.
- 사실 답변은 출처 마커를 달고 있고, 갭은 채워지는 대신 정직하게 보고됩니다.

그것이 가동된, 개인적인 하네스입니다. 여기서부터는 쓸수록 더 유용해집니다 — 당신의 운영
데이터는 제자리에 머무는 동안, 교훈이 축적되기 때문입니다.
