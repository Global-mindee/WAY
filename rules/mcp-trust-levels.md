# rules/mcp-trust-levels.md

> **목적**: CLAUDE.md 8. MCP 출처별 신뢰도 매핑의 상세 운영 규칙
> **적용 범위**: Layer 3 MCP 외부 시스템(DB·캘린더·메일·문서저장소·외부 API 등) 사용 시
> **버전**: v1.0
> **출처**: `harness-blueprint.md` Item 5 + 영역 C + `CLAUDE.md` 8.

---

## 1. 주요 용어

- **MCP**: Model Context Protocol — Layer 3 외부 시스템 접근 통로.
- **신뢰도 등급**: 3단계 (High / Medium / Low).
- **출처 인용**: 답변 본문에 MCP 출처와 등급을 명시하는 표기 의무.
- **AP3**: Blueprint 안티원칙 3 — "데이터는 능동적으로 격리되지 않는다" → 신뢰도 자동 약화 금지.

---

## 2. 3단계 등급 정의 (매핑 흐름)

연결한 MCP를 **출처의 통제권·검증 가능성** 기준으로 3단계에 매핑합니다. 핵심 판단축은 "이 데이터를 누가 입력·발신했는가"입니다.

| 등급 | 의미 | 판단 기준 |
|------|------|-----------|
| **High** | 본인(운영자)이 직접 입력·작성·발신한 자료 + 자사 운영 DB 전체 | 통제권이 운영자에게 있고 검증 가능 |
| **Medium** | 외부에서 공유받은 자료 + 외부 API 결과 | 통제권이 외부에 있으나 출처가 식별됨 |
| **Low** | 자동 발송·미확인 발신자 메시지 | 출처 미확인 또는 자동화 출력 |

### 산업·환경별 기본값 예시 (템플릿)

연결 MCP를 처음 매핑할 때 출발점으로 쓰는 일반 기본값입니다. 구체 등급은 7. 결재 흐름으로 확정합니다.

| 출처 유형(예시) | 기본 등급 | 사유 |
|----------------|----------|------|
| 자사 운영 DB (직접 운영) | **High** | 본인 운영 데이터, 검증 가능 |
| 캘린더·문서저장소 — 본인 직접 작성 항목 | **High** | 직접 작성 |
| 캘린더·문서저장소 — 외부에서 공유받은 항목 | **Medium** | 외부 입력 |
| 메일 — 본인 발신 | **High** | 직접 발신 |
| 메일 — 수신함, 알려진 발신자 | **Medium** | 외부 발신, 발신자 식별됨 |
| 외부 데이터·검색 API (항공·지도·날씨 등) | **Medium** | 외부 API |
| 메일 — 수신함, 미확인 발신자 / 스팸 | **Low** | 발신자 미확인 |
| 자동 발송 메시지 (시스템·봇) | **Low** | 자동화 출력 |

---

## 3. 출처 인용 표기 의무

답변에 MCP 데이터를 사용하면 등급별로 표기:

| 등급 | 표기 | 예시 |
|------|------|------|
| High | 표기 생략 가능 | `(출처: 자사 DB)` 또는 단순 인용 |
| **Medium** | **표기 의무** | `(출처: 공유받은 문서, 신뢰도: Medium)` |
| **Low** | **표기 의무 + 사유 명시** | `(출처: 수신 메일, 신뢰도: Low — 발신자 미확인)` |

---

## 4. Low 신뢰도 — UNCERTAIN 매핑

Low 등급 출처를 사실 단정에 사용할 때:
- 인라인 `[UNCERTAIN]` 마커 부착 (`rules/unknown-modes.md` 8.)
- 답변 끝 5모드 요약에 UNCERTAIN 건수 표기

예:
```
"내일 14시 미팅 예정" [UNCERTAIN] (출처: 수신 메일, 신뢰도: Low — 자동 발송)
```

---

## 5. 출처 충돌 시 처리

같은 사실에 두 MCP 출처가 충돌하면:
1. **두 출처 모두 명시** (어느 한쪽 임의 선택 금지)
2. `rules/unknown-modes.md` PARTIAL 매핑
3. Critical 작업이면 자동 Best-of-N 진입 (`rules/critical-path.md`)
4. `logs/operations.log` 에 `[timestamp] mcp-conflict: 출처A vs 출처B, 작업="..."` 기록

본 로그는 Blueprint 영역 D 메타 피드백 임계치(MCP 출처 충돌 > 5건/월) 분석의 입력.

---

## 6. 신뢰도 변경 정책 — 정적 + 명시만 (AP3 정합)

- 한 번 정해진 등급은 **능동적 자동 약화 금지**
- 충돌 누적 시에도 자동 강등 안 함
- 변경은 **운영자 명시 결정만** 가능 (시간·신호 기반 자동 약화 없음)
- 충돌 통계는 P6 메타 피드백에서 제공만, 변경 권고안 자동 작성 안 함

## 7. 신규 MCP 추가 흐름

**신규 MCP 연결 체크리스트** (미매핑 MCP 재발 방지):
- [ ] (1) 연결 즉시 출처 유형 분석 → 등급 제안
- [ ] (2) `decisions/pending.md` 결재 등록 (아래 양식)
- [ ] (3) 결재 후 8. 매핑 표 반영
- [ ] (4) SessionStart 스캔 훅이 "연결 MCP vs 매핑 표" diff로 미매핑 발견 시 경고 (회귀 방지)

신규 MCP 연결 시:
1. AI가 자동 매핑 시도 (출처 유형 분석 후 High/Medium/Low 제안)
2. `decisions/pending.md` 에 결재 요청 자동 등록:
   ```
   ## ☐ 항목 N: 신규 MCP 신뢰도 매핑
   - 범주: mcp-trust-mapping
   - MCP 이름: <예: 노트 앱>
   - AI 제안 등급: <Medium>
   - 근거: <자동 분석>
   - [ ] High 채택
   - [ ] Medium 채택 (제안 수락)
   - [ ] Low 채택
   ```
3. 운영자 결재 후 본 파일(`rules/mcp-trust-levels.md`)의 8. 매핑 표에 반영
4. MCP 해제 시 매핑 정보는 `rules/_archive/mcp-removed-<MCP명>.md` 로 이동 (never_delete)

## 8. 현재 매핑 표 (운영자 환경에 맞게 채움)

아래는 **표 구조의 예시 골격**입니다. 실제 연결한 MCP를 7. 흐름으로 결재한 뒤 이 표에 채웁니다.

| MCP(예시) | 등급 | 사유 |
|-----------|------|------|
| 자사 운영 DB | High | 본인 운영 DB |
| 캘린더 (본인 입력) | High | 직접 작성 |
| 캘린더 (본인 위임 작성) | **Medium** | 위임 대리 작성 (결재 반영) |
| 캘린더 (공유받은 항목) | Medium | 외부 입력 |
| 문서저장소 (본인 작성) | High | 직접 작성 |
| 문서저장소 (공유받은 항목) | Medium | 외부 입력 |
| 메일 (본인 발신) | High | 직접 발신 |
| 메일 (본인 위임 발신) | **Medium** | 위임 대리 발신 (결재 반영) |
| 메일 (수신, 알려진 발신자) | Medium | 외부 발신 |
| 메일 (수신, 미확인 발신자) | Low | 발신자 미확인 |
| 메일 (스팸) | Low | 자동 발송 가능성 |
| 메일 (자동 발송 메시지) | Low | 자동화 출력 |
| 외부 데이터·검색 API | Medium | 외부 API |
| WEB (기사·뉴스·논문) | **High** | 결재 반영 |
| WEB (유튜브·SNS) | **Medium** | 결재 반영 |
| WEB (그 외 default) | **Medium** | 결재 반영 |

미매핑 MCP는 7. 흐름에 따라 결재 후 추가합니다.

## 9. 예시 시나리오

**시나리오 A — High 신뢰도 단일 출처**
- 입력: "이번 달 매출 보여줘"
- 출처: 자사 DB 직접 쿼리
- 표기: `2026-05 매출 = ... (출처: 자사 DB)`

**시나리오 B — Medium + 표기 의무**
- 입력: "협력사 발송 자료 요약"
- 출처: 문서저장소에서 공유받은 PDF
- 표기: `요약 본문... (출처: 공유 문서, 신뢰도: Medium)`

**시나리오 C — Low + UNCERTAIN 매핑**
- 입력: "방금 받은 메일의 일정 정리해줘"
- 출처: 미확인 발신자 메일
- 표기: `일정 본문... [UNCERTAIN] (출처: 수신 메일, 신뢰도: Low — 발신자 미확인)`

**시나리오 D — 충돌**
- 입력: "다음 주 화요일 미팅"
- 캘린더 본인 입력: "팀 미팅 14:00"
- 자동 발송 메일: "팀 미팅 15:00"
- 표기: PARTIAL 매핑 + 양쪽 명시 + Critical 진입 시 Best-of-N

---

## 10. 결재 이력 (예시 골격)

- 위임 등급(캘린더·메일 위임 발신·작성) → **Medium** 으로 확정한 결재 기록을 남깁니다.
- WEB 도메인별 등급 정식화 결재 기록을 남기고 8. 표에 통합합니다.
- **MCP 자동 약화 임계치**: AP3와 정합하나, 임계치 표준화는 별도 결재 대상으로 남겨둡니다.
