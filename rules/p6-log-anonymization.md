# rules/p6-log-anonymization.md — P6 로그 익명화 규약

> **목적**: 하네스 로그·문서에 운영 데이터(제3자 실명·고객·위치·매출 등)가 유입되는 것을 차단 (CLAUDE.md 12절 단방향 원칙의 집행 규칙)
> **신설 배경**: 거버넌스 결정 (전수 감사 보안 항목 해소)
> **근거**: 전수 감사에서 operations.log에 인터뷰 대상 기업 실명·위치가 반복 유입되는 패턴 발견(push 시 원격 저장소 유출 경로)

---

## 1. 토큰화 규칙 — 기록 전 치환

하네스 로그(`logs/`)·문서에 다음이 들어가면 **기록 전 토큰으로 치환**한다:

| 대상 | 토큰 형식 | 예 |
|------|----------|-----|
| 제3자 기업·기관 실명 | `대상사A`·`대상사B` … (세션 내 일관) | (특정 기업명) → 대상사A |
| 인터뷰·현장 식별 | `현장L`·`int-NN` | (특정 건물·층) → 현장L |
| 제3자 개인 실명 | 역할명 | (특정 인명) 기획자 → 동료기획자 |
| 위치·주소 | 일반화 | 특정 층/동 → (생략) |
| 매출·결제·계약 금액 | **하네스 반입 금지** — 외부 레포로 | (logs 기록 불가) |

프로젝트 식별자(project-alpha 등)·이벤트 유형·통계는 메타 정보이므로 그대로 유지(P6 허용).

## 2. archive 경계

`decisions/archive/`에는 결재 **결정·사유 요약 + 외부 레포 포인터**만 보관한다. 사업 전략 전문·출판사 실명·매출 트리거 수치는 해당 프로젝트 외부 레포로 (하네스엔 "결정했다"는 메타만).

## 3. 본인(운영자) 식별정보

`self/self-definition.md`의 사업자번호·신고번호 등 본인 식별정보는 자동 로드 정체성 파일로 by-design 허용(레포 PRIVATE 완충). 단 **제3자** 정보는 quarantine 대상(거버넌스 결정). git 히스토리 기존 잔존분의 정비는 별도 비가역 작업(filter-repo)으로 결재 후 진행.

## 4. 적용 시점

- 신규 로그 기록: 본 규약 즉시 적용 (full-loop S8·CLAUDE.md 13절 2항 기록 시 점검)
- 기존 라인: 발견 시 추상화(소급 정비 진행). 전수 정비는 점진적
- self-audit.sh(차기 구현)에 P6 금칙어 grep 체크 포함 — 회귀 방지
