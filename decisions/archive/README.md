# decisions/archive/

> **목적**: 결재 통과(또는 거절·보류 종결) 항목의 영구 보관소
> **정책**: never_delete

---

## 이 폴더가 무엇인가

`decisions/pending.md`에서 응답이 완료된 결재 항목은 여기로 이동합니다.

- **파일명 규약**: `{YYYY-MM-DD}-{slug}.md` (예: `2026-06-10-batch-approvals.md`)
- **이동 시점**: 사용자가 체크박스로 응답을 마친 직후.

---

## never_delete의 의미

결재 결정은 **삭제하지 않습니다.** 과거 어떤 결정이 왜 내려졌는지(근거·옵션·선택)가 하네스의 의사결정 추적성(audit trail)을 이룹니다. 폐기된 결정도 흔적은 영구 보존됩니다.

`.gitkeep`은 빈 폴더 추적을 위해 유지합니다 (삭제 금지).
